The Human Genome Variation Society (HGVS) nomenclature has become the defacto standard for describing sequence variants in clinical reports and publications. Conversion of HGVS to and from genomic coordinates is thus a crucial requirement for genomics.

RefSeq and Ensembl transcripts are versioned (eg NM_153253.3 is version 3). Exon coordinates can change between versions, so to correctly convert historical HGVS data, you may need many different transcript versions.

We provide a REST web API for retrieving RefSeq and Ensembl transcript version data https://github.com/SACGF/cdot_rest

This is hosted here: http://cdot.cc/ - though we have obtained new url cdotlib.org - will move it there soon 

And have written integration code for the two most popular Python HGVS libraries: BioCommons HGVS and Counsyl pyHGVS.

This increases the number of transcript versions resolvable by Python from 141k to 1.3M (need to get exact figures)

Also support Telomere to telomere HGVS resolution (is this a first?)

A long time ago, someone wrote gtf-to-genes which used a binary file for rapid loading - but gzipped JSON is faster (and human readable). Discontinued.
